Helix Vault

Embedded Systems Final Project — ESE 5190 (Fall 2025)

Team: Byte This (Team 16)
Members: Yongwoo Park · Jeevan Karandikar · Tomas Ascoli

A three layer secure safe integrating biometric authentication, analog combination input, and PIN verification. Powered by two ATmega328PB microcontrollers and an ESP32 fingerprint module, Helix Vault blends embedded security logic, mechanical actuation, and a guided LCD user interface.

System Overview

Helix Vault uses a sequential three layer unlock flow:

  1. Fingerprint verification (ESP32 + R503 sensor)
  2. Analog combination using potentiometers + switches (ADC readings)
  3. PIN entry using a matrix keypad

A DC motor opens the sliding door after fingerprint recognition, revealing the analog combination and keypad stage.

SRS-01 — Low Power Management

Not implemented, but system restart never compromises security.

SRS-02 — Correct Password Recognition

100% success across all tests — no incorrect fingerprint, combination, or PIN ever bypassed the system.

SRS-03 — Servo + Motor Control

DC motor opens sliding door only after valid fingerprint. Servo latch opens only after valid PIN. Fully validated.

SRS-04 — System Lockdown

Implemented manual lockdown ("*" key).

SRS-05 — Storage Management

Credential modification handled via serial interface.

Conclusion

What Went Well

  • Multi-MCU GPIO communication
  • LCD-guided UI
  • Consistent fingerprint performance

Lessons Learned

  • UART debugging requires careful electrical design
  • Mechanical torque can bottleneck software progress
  • Modular firmware architecture accelerates integration

Next Steps

  • Add facial or iris recognition
  • Custom PCB for compact assembly
  • Add override key
  • Logging + timestamped unlock events
